# Gridix Core Flows and Invariants

## Purpose

这份文档把 Gridix v5.0.0 收口阶段的“核心功能不能坏”落成工程边界。  
后续任何修复、重构或 UI 调整，只要触碰这里列出的功能流或不变量，就必须先证明没有破坏对应边界；否则应拒绝合入。

## Practical Use

- 后续修复如果触碰核心流，PR 或 recovery note 必须点名“改的是哪条功能流、保护的是哪条不变量”。
- 如果某个改动无法清楚说明它如何保持这些不变量，默认应视为高风险改动。
- 下一批 recovery 文档应优先补足仍然薄弱的验证面：`cancel vs stale`、dialog 溢出、ER 顶层交互、grid save 端到端。
